ğŸ“Œ Layer Explanation

  • **Core**: Contains global utilities like themes, constants, and error definitions that can be used by any layer.
  • **Data**: The external layer that handles data fetching from sources like APIs, Firebase, or a local database. It also includes data models for serialization.
  • **Domain**: The heart of your app's business logic. It's a pure Dart layer that contains entities (data classes), abstract repository contracts, and use cases (business logic). It has no knowledge of the UI or data sources.
  • **Presentation**: The UI layer. It contains your widgets, screens, and state management logic (e.g., Provider, Bloc). It knows about the `domain` layer but not the `data` layer.

ğŸ“Œ Example Repository Contract

The `domain` layer defines an abstract contract that the `data` layer must implement. This is a crucial part of the architecture, as it decouples your business logic from the data source.

domain/repositories/post_repository.dart 

abstract class PostRepository {
  Future> getPosts();
}

ğŸ“Œ Example Repository Implementation

The `data` layer implements the contract defined in the `domain` layer. This is where you write the actual code for fetching data, like making an API call with Dio.

data/repositories/post_repository_impl.dart 

import 'package:dio/dio.dart';
import '../../domain/repositories/post_repository.dart';
import '../models/post_model.dart';

class PostRepositoryImpl implements PostRepository {
  final Dio dio;
  PostRepositoryImpl(this.dio);

  @override
  Future> getPosts() async {
    final response = await dio.get('https://jsonplaceholder.typicode.com/posts');
    return (response.data as List).map((e) => PostModel.fromJson(e)).toList();
  }
}

ğŸ“Œ Example Use Case

Use cases contain your business logic. They orchestrate the flow of data and perform operations without knowing where the data comes from. They simply call the abstract repository interface.

domain/usecases/get_posts.dart 

import '../repositories/post_repository.dart';
import '../entities/post.dart';

class GetPosts {
  final PostRepository repository;
  GetPosts(this.repository);

  Future> call() async {
    return await repository.getPosts();
  }
}

ğŸ“Œ Example UI

The UI layer only interacts with the `domain` layer (the use cases). It never directly calls an API or a database. This keeps the UI completely independent of the data source.

presentation/pages/post_page.dart 

import 'package:flutter/material.dart';
import '../../domain/usecases/get_posts.dart';

class PostPage extends StatelessWidget {
  final GetPosts getPosts;
  PostPage(this.getPosts);

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(title: Text("Posts")),
      body: FutureBuilder(
        future: getPosts(),
        builder: (context, snapshot) {
          if (snapshot.connectionState == ConnectionState.waiting) {
            return Center(child: CircularProgressIndicator());
          }
          if (snapshot.hasError) return Text("Error: ${snapshot.error}");
          final posts = snapshot.data ?? [];
          return ListView.builder(
            itemCount: posts.length,
            itemBuilder: (_, i) => ListTile(title: Text(posts[i].title)),
          );
        },
      ),
    );
  }
}
