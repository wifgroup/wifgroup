Day 19: CRUD with Firebase Firestore

Learn to create, read, update, and delete data with Firebase Firestore to build dynamic, data-driven apps. üìù

üß† What is Firestore?

Firestore is a flexible, scalable **NoSQL cloud database** from Firebase. It stores data in a hierarchical structure of **collections** and **documents**. It‚Äôs perfect for real-time applications because it automatically keeps your data in sync across all connected devices.

  • Stores data in **collections ‚Üí documents ‚Üí fields**.
  • Provides **real-time updates** to connected devices.
  • Highly scalable and easy to integrate.

üì¶ Step 1: Firestore Dependency

Make sure you have the `cloud_firestore` package in your `pubspec.yaml` file from Day 17. If not, add it now.

pubspec.yaml 

dependencies:
  flutter:
    sdk: flutter
  cloud_firestore: ^5.6.0
  firebase_core: ^2.15.0

Run `flutter pub get` to fetch the package.

üìå Step 2: Initialize Firestore

To use Firestore, you need an instance of `FirebaseFirestore`. You can create a global instance like this:

Initialize Firestore 

import 'package:cloud_firestore/cloud_firestore.dart';

FirebaseFirestore firestore = FirebaseFirestore.instance;

üìù Step 3: Create Data

To add a new document to a collection, use the `.add()` method. Firestore will automatically generate a unique ID for the new document.

Create a new document 

await firestore.collection('tasks').add({
  'title': 'Buy groceries',
  'description': 'Milk, Eggs, Bread',
  'completed': false,
});

üìñ Step 4: Read Data

To read data in real-time, use `StreamBuilder` and listen to a collection's `.snapshots()`. The builder function will automatically rebuild whenever data changes in the Firestore database.

Read data with StreamBuilder 

StreamBuilder(
  stream: firestore.collection('tasks').snapshots(),
  builder: (context, snapshot) {
    if (!snapshot.hasData) return CircularProgressIndicator();

    final tasks = snapshot.data!.docs;

    return ListView.builder(
      itemCount: tasks.length,
      itemBuilder: (context, index) {
        final task = tasks[index];
        return ListTile(
          title: Text(task['title']),
          subtitle: Text(task['description']),
          trailing: Checkbox(
            value: task['completed'],
            onChanged: (value) {
              firestore.collection('tasks').doc(task.id).update({
                'completed': value,
              });
            },
          ),
        );
      },
    );
  },
)

The `task.id` is the unique document ID, which is essential for updating or deleting specific documents.

‚úèÔ∏è Step 5: Update Data

To update an existing document, you need its document ID. Use the `.doc()` method to specify the document and then call `.update()` with the new data.

Update a document 

await firestore.collection('tasks').doc('DOCUMENT_ID').update({
  'title': 'Buy groceries and fruits',
});

‚ùå Step 6: Delete Data

To delete a document, you also need its ID. Use `.doc()` to target the document, and then call `.delete()`.

Delete a document 

await firestore.collection('tasks').doc('DOCUMENT_ID').delete();

üßë‚Äçüíª Example: Simple To-Do App

This simple app demonstrates all CRUD operations in action. It creates a task, displays a list of tasks in real-time, and allows you to update and delete them.

Complete Code 

import 'package:flutter/material.dart';
import 'package:cloud_firestore/cloud_firestore.dart';

class TodoApp extends StatelessWidget {
  final FirebaseFirestore firestore = FirebaseFirestore.instance;
  final TextEditingController _controller = TextEditingController();

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(title: Text("Firestore CRUD")),
      body: Column(
        children: [
          Padding(
            padding: EdgeInsets.all(8),
            child: TextField(controller: _controller, decoration: InputDecoration(labelText: 'New Task')),
          ),
          ElevatedButton(
            onPressed: () async {
              await firestore.collection('tasks').add({
                'title': _controller.text,
                'completed': false,
              });
              _controller.clear();
            },
            child: Text("Add Task"),
          ),
          Expanded(
            child: StreamBuilder(
              stream: firestore.collection('tasks').snapshots(),
              builder: (context, snapshot) {
                if (!snapshot.hasData) return CircularProgressIndicator();
                final tasks = snapshot.data!.docs;
                return ListView.builder(
                  itemCount: tasks.length,
                  itemBuilder: (context, index) {
                    final task = tasks[index];
                    return ListTile(
                      title: Text(task['title']),
                      trailing: Row(
                        mainAxisSize: MainAxisSize.min,
                        children: [
                          IconButton(
                            icon: Icon(Icons.edit),
                            onPressed: () async {
                              await firestore.collection('tasks').doc(task.id).update({'title': 'Updated Task'});
                            },
                          ),
                          IconButton(
                            icon: Icon(Icons.delete),
                            onPressed: () async {
                              await firestore.collection('tasks').doc(task.id).delete();
                            },
                          ),
                        ],
                      ),
                    );
                  },
                );
              },
            ),
          ),
        ],
      ),
    );
  }
}
