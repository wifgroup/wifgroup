Day 14: Responsive Design in Flutter

Learn how to make your apps adapt beautifully to different screen sizes and orientations. üì±üíª

üßê Why Responsive Design?

A UI that looks great on all devices is essential for a professional app. Responsive design helps you build a single codebase that works everywhere, preventing common issues and improving user experience.

  • A UI that looks **great on all devices** (small phones ‚Üí tablets ‚Üí desktops).
  • Prevents overflow issues (RenderFlex overflowed‚Ä¶).
  • Improves usability & accessibility.

üìå MediaQuery ‚Äì Getting Device Info

MediaQuery gives details about the current device screen. It's a great tool for making minor adjustments like font sizes or padding based on the screen's dimensions.

MediaQuery Demo 

import 'package:flutter/material.dart';

class MediaQueryDemo extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    final size = MediaQuery.of(context).size; // screen width & height
    final isPortrait = MediaQuery.of(context).orientation == Orientation.portrait;

    return Scaffold(
      appBar: AppBar(title: Text("MediaQuery Demo")),
      body: Center(
        child: Text(
          "Width: ${size.width}, Height: ${size.height}\nOrientation: ${isPortrait ? "Portrait" : "Landscape"}",
          textAlign: TextAlign.center,
        ),
      ),
    );
  }
}

üëâ Use MediaQuery.of(context).size.width to create **responsive layouts**.

üìå Example: Responsive Text Size

This snippet shows how to make a font size scale proportionally with the screen's width.

Responsive Text Example 

Text(
  "Responsive Text",
  style: TextStyle(fontSize: MediaQuery.of(context).size.width * 0.05),
);

This ensures the font **scales with screen width**.

üõ†Ô∏è LayoutBuilder ‚Äì Building Based on Constraints

Unlike MediaQuery, which gives the screen size, **LayoutBuilder** tells you the **size of the widget's parent**. This is perfect for adapting layouts inside a specific area of your app.

LayoutBuilder Demo 

class LayoutBuilderDemo extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(title: Text("LayoutBuilder Demo")),
      body: LayoutBuilder(
        builder: (context, constraints) {
          if (constraints.maxWidth < 600) {
            return Center(child: Text("üì± Mobile Layout"));
          } else {
            return Center(child: Text("üíª Tablet/Desktop Layout"));
          }
        },
      ),
    );
  }
}

üëâ LayoutBuilder is perfect for **adaptive UIs** where different widgets are used based on screen width.

üéØ Combining MediaQuery + LayoutBuilder

For truly robust responsive design, you can use a combination of both to create distinct layouts for different device types.

Combining both widgets 

class ResponsiveHome extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    return LayoutBuilder(
      builder: (context, constraints) {
        if (constraints.maxWidth < 600) {
          return _mobileLayout();
        } else if (constraints.maxWidth < 1200) {
          return _tabletLayout();
        } else {
          return _desktopLayout();
        }
      },
    );
  }

  Widget _mobileLayout() => Scaffold(body: Center(child: Text("üì± Mobile")));
  Widget _tabletLayout() => Scaffold(body: Center(child: Text("üíª Tablet")));
  Widget _desktopLayout() => Scaffold(body: Center(child: Text("üñ•Ô∏è Desktop")));
}
