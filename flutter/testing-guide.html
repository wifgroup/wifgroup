Day 29: Testing (Unit, Widget, Integration)

Learn to write and run tests to ensure your apps are reliable, robust, and bug-free. üß™

üß† Why Testing Matters

Yesterday, we optimized apps for performance. Today, we'll ensure our apps are reliable by adding automated tests. Testing is a crucial part of the software development lifecycle that prevents regressions, ensures features work as expected, and ultimately gives you confidence in your code before a release.

  • Prevents regressions when code changes.
  • Ensures features work as expected.
  • Improves code quality and confidence before release.
  • Saves debugging time in production.

üß© Types of Tests in Flutter

Flutter provides a testing framework that supports three main types of tests, each with a different focus and speed.

Test Type What it Covers Speed
Unit Test Small pieces of logic (functions, classes) ‚ö° Fast
Widget Test Individual widgets in isolation ‚ö° Medium
Integration Test Full app, real user flow üê¢ Slower

‚úÖ 1. Unit Testing

Unit tests verify the behavior of a single class or function. They are fast to run and provide quick feedback on the correctness of your code's logic.

test/counter_test.dart 

import 'package:flutter_test/flutter_test.dart';

int add(int a, int b) => a + b;

void main() {
  test('adds two numbers correctly', () {
    expect(add(2, 3), 5);
  });
}

To run this test, use the command: `flutter test`.

‚úÖ 2. Widget Testing

Widget tests build and interact with a single widget in a controlled environment, ensuring that the UI looks and behaves as expected. You can simulate user gestures like taps and scrolls without running the app on a device or emulator.

test/widget_test.dart 

import 'package:flutter/material.dart';
import 'package:flutter_test/flutter_test.dart';

void main() {
  testWidgets('Counter increments when button is tapped',
      (WidgetTester tester) async {
    // Build widget
    await tester.pumpWidget(MaterialApp(
      home: Scaffold(
        body: StatefulBuilder(
          builder: (context, setState) {
            int counter = 0;
            return Column(
              children: [
                Text('Count: $counter', key: Key('counter')),
                ElevatedButton(
                  onPressed: () => setState(() => counter++),
                  child: Text('Increment'),
                ),
              ],
            );
          },
        ),
      ),
    ));

    // Verify initial state
    expect(find.text('Count: 0'), findsOneWidget);

    // Tap button
    await tester.tap(find.text('Increment'));
    await tester.pump();

    // Verify updated state
    expect(find.text('Count: 1'), findsOneWidget);
  });
}

‚úÖ 3. Integration Testing

Integration tests run on a real device or emulator and verify the entire app's functionality from end to end. They simulate a real user's journey through the app, ensuring all components work together seamlessly.

integration_test/app_test.dart 

import 'package:flutter_test/flutter_test.dart';
import 'package:integration_test/integration_test.dart';
import 'package:my_app/main.dart' as app;

void main() {
  IntegrationTestWidgetsFlutterBinding.ensureInitialized();

  testWidgets('App startup and button test', (tester) async {
    app.main();
    await tester.pumpAndSettle();

    // Find button
    final button = find.text('Increment');
    expect(button, findsOneWidget);

    // Tap button
    await tester.tap(button);
    await tester.pumpAndSettle();

    // Check updated counter
    expect(find.text('1'), findsOneWidget);
  });
}

To run integration tests, use: `flutter test integration_test`.

üõ†Ô∏è Testing Tools

  • `flutter_test` ‚Üí The core package for unit and widget tests.
  • `integration_test` ‚Üí The official package for end-to-end testing.
  • `mockito` or `mocktail` ‚Üí Essential packages for creating mock objects to test code in isolation.
  • **CI/CD pipelines** (e.g., GitHub Actions, Bitrise) ‚Üí Automate your tests with every code change to catch bugs early.

‚úÖ Checklist for Today

  • Write at least 1 unit test
  • Write at least 1 widget test
  • Run an integration test on an emulator/device
  • Add tests to a CI/CD pipeline (optional)
