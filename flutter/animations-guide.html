Day 22: Animations in Flutter

Learn to bring motion and interactivity into your apps with implicit, explicit, and Hero animations. üé®

üß† Why Animations Matter?

Yesterday, we learned about error handling, logging, and debugging. Today, we'll focus on bringing our apps to life with motion. Animations aren't just for flair; they make apps more fun, dynamic, and user-friendly by guiding user focus and improving the overall user experience.

  • Draw user attention to important elements ‚ú®
  • Make transitions smoother (page changes, button clicks)
  • Improve usability & aesthetics üé®

üìå Step 1: Implicit Animations

Implicit animations are the easiest way to get started with motion in Flutter. Widgets like `AnimatedContainer` and `AnimatedOpacity` automatically animate a property change when you update its value. This is a great way to add smooth transitions without writing complex animation logic.

Example: AnimatedContainer

This example shows how a container's size and color smoothly change when a button is pressed.

AnimatedContainer Demo 

import 'package:flutter/material.dart';

class AnimatedContainerDemo extends StatefulWidget {
  @override
  _AnimatedContainerDemoState createState() => _AnimatedContainerDemoState();
}

class _AnimatedContainerDemoState extends State {
  bool _isBig = false;

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      body: Center(
        child: AnimatedContainer(
          width: _isBig ? 200 : 100,
          height: _isBig ? 200 : 100,
          color: _isBig ? Colors.blue : Colors.red,
          duration: Duration(seconds: 1),
          curve: Curves.easeInOut,
        ),
      ),
      floatingActionButton: FloatingActionButton(
        onPressed: () => setState(() => _isBig = !_isBig),
        child: Icon(Icons.play_arrow),
      ),
    );
  }
}

üëâ Tap the `FloatingActionButton` to see the container smoothly resize and change color.

üìå Step 2: Tween Animations (Explicit Animations)

For more fine-grained control, you can use explicit animations with an `AnimationController` and a `Tween`. This gives you full control over the animation's timing, duration, and values.

Tween Animation Demo 

import 'package:flutter/material.dart';

class TweenAnimationDemo extends StatefulWidget {
  @override
  _TweenAnimationDemoState createState() => _TweenAnimationDemoState();
}

class _TweenAnimationDemoState extends State
    with SingleTickerProviderStateMixin {
  late AnimationController _controller;
  late Animation _animation;

  @override
  void initState() {
    super.initState();
    _controller =
        AnimationController(vsync: this, duration: Duration(seconds: 2));
    _animation = Tween(begin: 0, end: 200).animate(_controller);
    _controller.repeat(reverse: true);
  }

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      body: Center(
        child: AnimatedBuilder(
          animation: _animation,
          builder: (_, child) {
            return Container(
              width: _animation.value,
              height: _animation.value,
              color: Colors.purple,
            );
          },
        ),
      ),
    );
  }

  @override
  void dispose() {
    _controller.dispose();
    super.dispose();
  }
}

üëâ Here, a square smoothly grows and shrinks in a continuous loop. The `AnimatedBuilder` widget is used to rebuild only the animated part of the UI, improving performance.

üìå Step 3: Hero Animations (Page Transitions)

A **Hero** animation creates a dramatic visual effect by smoothly transitioning a shared widget between two different screens. It's a great way to link two screens and provide visual continuity.

Hero Animation Example 

// First Page
GestureDetector(
  onTap: () => Navigator.push(
    context,
    MaterialPageRoute(builder: (_) => SecondPage()),
  ),
  child: Hero(
    tag: "profile-pic",
    child: CircleAvatar(
      radius: 40,
      backgroundImage: AssetImage("assets/profile.jpg"),
    ),
  ),
);

// Second Page
Hero(
  tag: "profile-pic",
  child: Image.asset("assets/profile.jpg", width: 200, height: 200),
);

üëâ The image smoothly transitions from a small circle avatar on the first screen into a larger image on the second screen.

üìå Step 4: Curves

Animation **Curves** define the pace and feel of an animation. Using different curves can drastically change the user's perception of the motion. Try experimenting with these to find the perfect effect:


Curves.linear  // Constant speed
Curves.easeIn  // Starts slow, speeds up
Curves.easeOut // Starts fast, slows down
Curves.bounceOut // Bounces at the end
Curves.elasticInOut // An elastic effect at the beginning and end
