Day 26: API Integration in Flutter

Learn to fetch data from REST APIs and power your app with real-time information. üåê

üß† Key Concepts

Yesterday, we explored Local Storage. Today, we'll connect our Flutter apps to the internet by integrating **REST APIs**. APIs are essential for modern apps, allowing them to fetch real-time data, send user information, and connect with various external services.

  • **HTTP**: The protocol for communication over the web.
  • **REST API**: A standard way to send and receive data, typically in JSON format.
  • **JSON**: A lightweight, human-readable data format.
  • **http package**: A simple, beginner-friendly package for making network requests.
  • **Dio package**: A powerful, feature-rich alternative to `http` that's great for complex, production-level apps.

üìå Step 1: Setup Dependencies

To get started, add either the `http` package or the `dio` package to your `pubspec.yaml` file. For this lesson, we will install both to show the difference.

pubspec.yaml 

dependencies:
  http: ^1.2.2
  dio: ^5.4.0

Run `flutter pub get` to fetch the packages.

üìå Step 2: Using `http`

The `http` package is a great, simple choice for basic GET, POST, PUT, and DELETE requests. It provides a straightforward API that's easy to understand for beginners.

Using http to fetch users 

import 'dart:convert';
import 'package:http/http.dart' as http;

Future fetchUsers() async {
  final url = Uri.parse('https://jsonplaceholder.typicode.com/users');
  final response = await http.get(url);

  if (response.statusCode == 200) {
    final List data = jsonDecode(response.body);
    print("Users: $data");
  } else {
    print("Error: ${response.statusCode}");
  }
}

üëâ Use `http` for simple requests where you don't need advanced features.

üìå Step 3: Using Dio (Advanced API Client)

Dio is a more powerful HTTP client that includes features like interceptors, form data support, request cancellation, and robust error handling. It's often the preferred choice for production apps.

Using Dio to fetch posts 

import 'package:dio/dio.dart';

final dio = Dio();

Future fetchPosts() async {
  try {
    final response = await dio.get('https://jsonplaceholder.typicode.com/posts');
    print(response.data);
  } catch (e) {
    print("Error: $e");
  }
}

Add Headers & Interceptors

Dio's interceptors are particularly useful for adding headers like authorization tokens or for logging requests globally.

Add headers and interceptors 

dio.options.headers['Authorization'] = 'Bearer YOUR_TOKEN';

dio.interceptors.add(InterceptorsWrapper(
  onRequest: (options, handler) {
    print("Request: ${options.path}");
    return handler.next(options);
  },
  onError: (e, handler) {
    print("Error: ${e.message}");
    return handler.next(e);
  },
));

üëâ Use Dio for production apps that require features like logging, retries, and interceptors.

üìå Step 4: Parsing JSON into Models

It's a best practice to convert raw JSON data into strongly-typed Dart objects (models). This makes your code safer and easier to work with. You can create a factory constructor `fromJson` to handle this.

User model with fromJson 

class User {
  final int id;
  final String name;
  final String email;

  User({required this.id, required this.name, required this.email});

  factory User.fromJson(Map json) {
    return User(
      id: json['id'],
      name: json['name'],
      email: json['email'],
    );
  }
}

Usage:

Converting JSON to a list of models 

final List data = jsonDecode(response.body);
final users = data.map((u) => User.fromJson(u)).toList();
