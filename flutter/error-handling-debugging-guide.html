Day 21: Error Handling, Logging, and Debugging Tools

Learn how to make your apps stable and bug-free by catching errors and using powerful debugging tools. üõ†Ô∏è

üß† Why Error Handling & Logging Matters

Yesterday, we learned Firebase Storage and uploading images/files. Today, we'll focus on making your apps stable and bug-free. A professional developer knows how to prevent crashes, quickly identify issues, and understand unexpected behavior. This is where error handling, logging, and debugging tools become invaluable.

  • Prevent app crashes üî•
  • Track unexpected behavior
  • Improve user experience
  • Debug faster during development

üìå Step 1: Error Handling with Try/Catch

The most common way to handle predictable errors in Dart is with `try/catch` blocks. You wrap code that might fail in a `try` block and handle any exceptions in the `catch` block.

Basic try/catch 

try {
  int result = 100 ~/ 0; // Division by zero, this will throw an error
} catch (e) {
  print("Error occurred: $e");
}

You can also catch **specific exceptions**, which is useful for handling different types of errors with custom logic. For example, catching a `FirebaseAuthException` to show a user-friendly message.

Catching specific exceptions 

try {
  // Firebase Auth sign-in
  await FirebaseAuth.instance.signInWithEmailAndPassword(email: email, password: password);
} on FirebaseAuthException catch (e) {
  if (e.code == 'user-not-found') {
    print("No user found for that email");
  } else if (e.code == 'wrong-password') {
    print("Incorrect password");
  }
} catch (e) {
  print("Unexpected error: $e");
}

üìå Step 2: Using FlutterError for Widget Tree Errors

The `FlutterError.onError` handler is a global way to catch errors that occur within the Flutter framework itself. It's especially useful for catching UI-level errors that might not be caught by a standard `try/catch` block. You can also use this to send errors to logging services like Sentry or Firebase Crashlytics.

Using FlutterError.onError 

FlutterError.onError = (FlutterErrorDetails details) {
  FlutterError.presentError(details); // default error handling
  print("Caught FlutterError: ${details.exception}");
};

üìå Step 3: Logging with print & debugPrint

For simple logging, `print()` and `debugPrint()` are your go-to tools. `print()` can be useful but may truncate large logs. `debugPrint()` is a better alternative as it automatically formats and prevents truncation.

Using debugPrint 

debugPrint("Debug info: App started successfully");

üìå Step 4: Using the `logger` Package

For more advanced logging, the `logger` package provides a more structured and color-coded output. It's a great tool for categorizing your logs and can be easily integrated with other services.

First, add the dependency to `pubspec.yaml`:

pubspec.yaml 

dependencies:
  logger: ^1.2.2

Then, use it in your code:

Using the Logger 

import 'package:logger/logger.dart';

var logger = Logger();

logger.d("Debug message");
logger.i("Info message");
logger.w("Warning message");
logger.e("Error message");

üìå Step 5: Debugging Tools in Flutter

Flutter comes with a powerful suite of debugging tools. You can access these in your IDE (VS Code or Android Studio) or via a browser by running `flutter run --device-id web-server` and opening the link in your console.

  • **Flutter DevTools**: An all-in-one suite for inspecting your widget tree, profiling performance, and viewing network calls.
  • **Hot Reload & Hot Restart**: Use hot reload for fast UI changes without losing state, and hot restart for a full rebuild when you change the app state or logic.
  • **Breakpoints**: Pause code execution at a specific line to inspect variable values and understand the flow of your program.
  • **Flutter Inspector**: A tool within DevTools that visualizes your widget tree and allows you to inspect properties and rebuild widgets.

üìå Step 6: Handling Asynchronous Errors

When working with `Future`s or async functions, it's crucial to use `try/catch` to handle potential errors that might occur during network requests or file operations.

Handling async errors 

Future fetchData() async {
  try {
    var result = await someAsyncFunction();
    print(result);
  } catch (e) {
    print("Async error: $e");
  }
}
