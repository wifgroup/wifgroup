Day 23: Custom Paint, Drawing, Canvas Basics

Learn to draw custom shapes, paths, and graphics with Flutter's powerful painting engine. üñåÔ∏è

üß† Why Use Custom Paint?

Yesterday, we explored animations. Today, we'll dive into Flutter's painting engine to draw shapes, paths, and custom designs directly on the screen. This gives you the freedom to build custom graphics, charts, signature pads, loaders, and even game UIs. üïπÔ∏è

Use CustomPaint when:

  • Built-in widgets aren't flexible enough for your design.
  • You need to create custom charts, waves, or progress rings.
  • You want to build lightweight graphics without relying on image assets.

üìå Step 1: The Basics ‚Äì CustomPaint

The `CustomPaint` widget is a fundamental part of Flutter's painting system. It takes a `CustomPainter` class where you define your drawing logic using the `Canvas` API.

CustomPaint Demo 

import 'package:flutter/material.dart';

class CustomPaintDemo extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    return Scaffold(
      body: Center(
        child: CustomPaint(
          size: Size(200, 200),
          painter: CirclePainter(),
        ),
      ),
    );
  }
}

class CirclePainter extends CustomPainter {
  @override
  void paint(Canvas canvas, Size size) {
    final paint = Paint()
      ..color = Colors.blue
      ..style = PaintingStyle.fill;

    // Draw a circle
    canvas.drawCircle(Offset(size.width / 2, size.height / 2), 80, paint);
  }

  @override
  bool shouldRepaint(covariant CustomPainter oldDelegate) => false;
}

üëâ This code paints a solid blue circle in the center of a 200x200 area.

üìå Step 2: Drawing Shapes

The `Canvas` object provides methods for drawing basic shapes like rectangles, lines, and ovals.

Drawing different shapes 

// Rectangle
canvas.drawRect(Rect.fromLTWH(20, 20, 100, 150), paint);

// Line
paint.strokeWidth = 4;
canvas.drawLine(Offset(0, 0), Offset(200, 200), paint);

// Oval/Ellipse
canvas.drawOval(Rect.fromLTWH(50, 50, 120, 80), paint);

üìå Step 3: Paths (Custom Shapes)

For more complex shapes like triangles, stars, or intricate curves, you can use a `Path` object. A path is a series of connected points and lines that you define, which can then be drawn on the canvas.

Drawing a custom triangle with Path 

class TrianglePainter extends CustomPainter {
  @override
  void paint(Canvas canvas, Size size) {
    final paint = Paint()
      ..color = Colors.red
      ..style = PaintingStyle.fill;

    final path = Path();
    path.moveTo(size.width / 2, 0);          // Top point
    path.lineTo(0, size.height);             // Bottom left point
    path.lineTo(size.width, size.height);    // Bottom right point
    path.close();

    canvas.drawPath(path, paint);
  }

  @override
  bool shouldRepaint(covariant CustomPainter oldDelegate) => false;
}

üìå Step 4: Combining Paint Styles

The `Paint` object is where you define the visual properties of what you're drawing, like color, style (fill or stroke), and stroke width. You can easily switch between them to create different effects.

Changing paint styles 

paint
  ..color = Colors.green
  ..style = PaintingStyle.stroke
  ..strokeWidth = 5;

üìå Step 5: Advanced Example ‚Äì Wave Animation üåä

This example demonstrates how you can combine `Path` with a mathematical function (`sin()`) to create a dynamic, animated wave effect. You'd typically wrap this in an `AnimatedBuilder` to make it animate continuously.

WavePainter Class 

class WavePainter extends CustomPainter {
  @override
  void paint(Canvas canvas, Size size) {
    final paint = Paint()
      ..color = Colors.blueAccent
      ..style = PaintingStyle.fill;

    final path = Path();
    path.moveTo(0, size.height * 0.5);

    for (double i = 0; i <= size.width; i++) {
      path.lineTo(i, size.height * 0.5 + 20 * (i / 20).sin()); // wave effect
    }

    path.lineTo(size.width, size.height);
    path.lineTo(0, size.height);
    path.close();

    canvas.drawPath(path, paint);
  }

  @override
  bool shouldRepaint(covariant CustomPainter oldDelegate) => true;
}

üëâ Setting `shouldRepaint` to `true` ensures this widget rebuilds on every frame, which is necessary for animations.
