Day 25: Local Storage (SharedPreferences, Hive, SQLite)

Learn to save user preferences, structured data, and build offline-first apps in Flutter. üíæ

üß† Flutter Local Storage Options

Yesterday, we explored Background Services & Notifications. Today, we'll learn how to store data locally within a Flutter app. Local storage is crucial for saving user preferences, caching data for offline access, and improving app performance by reducing network requests. Choosing the right storage solution depends on your data's structure and complexity.

Storage Type Use Case Pros Cons
SharedPreferences Store key-value pairs (settings, flags) Simple, fast Only supports primitives (String, int, bool, double, List)
Hive Store structured NoSQL-like data Super fast, no native dependencies Not ideal for very complex relations
SQLite Relational database with full SQL support Full SQL support, complex queries Slower than Hive, more setup

üìå Step 1: SharedPreferences (Key-Value Storage)

This is the simplest form of local storage and is perfect for saving small amounts of data like user settings, preferences, or a login flag.

First, add the `shared_preferences` dependency to your `pubspec.yaml` file:

pubspec.yaml 

dependencies:
  shared_preferences: ^2.3.2

Usage:

Saving and Loading Data 

import 'package:shared_preferences/shared_preferences.dart';

Future saveData() async {
  final prefs = await SharedPreferences.getInstance();
  await prefs.setBool('isDarkMode', true);
}

Future loadData() async {
  final prefs = await SharedPreferences.getInstance();
  bool? isDarkMode = prefs.getBool('isDarkMode');
  print("Dark Mode: $isDarkMode");
}

üëâ Best for flags, login state, and small preferences.

üìå Step 2: Hive (NoSQL DB for Flutter)

Hive is a lightweight and blazing-fast NoSQL database that works great for storing structured data like tasks, notes, or user profiles. It's known for its speed and ease of use, with no native dependencies.

First, add the dependencies:

pubspec.yaml 

dependencies:
  hive: ^2.2.3
  hive_flutter: ^1.1.0

Then, initialize Hive in your `main()` function:

Initialize Hive in main() 

import 'package:hive_flutter/hive_flutter.dart';

void main() async {
  await Hive.initFlutter();
  await Hive.openBox('myBox');
}

Usage:

Using Hive Box 

var box = Hive.box('myBox');
box.put('username', 'Harshil');
print(box.get('username')); // Harshil

üëâ Best for apps needing offline caching, notes, and tasks.

üìå Step 3: SQLite (Relational DB)

For more complex data with structured relationships, a full relational database like SQLite is the best choice. The `sqflite` package provides a robust interface for interacting with the database.

First, add the dependencies:

pubspec.yaml 

dependencies:
  sqflite: ^2.4.0
  path: ^1.9.0

Example:

This example shows how to open a database, create a table, and perform a simple insert and query.

SQLite Example 

import 'package:sqflite/sqflite.dart';
import 'package:path/path.dart';

Future databaseExample() async {
  final db = await openDatabase(
    join(await getDatabasesPath(), 'demo.db'),
    onCreate: (db, version) {
      return db.execute(
        'CREATE TABLE users(id INTEGER PRIMARY KEY, name TEXT)',
      );
    },
    version: 1,
  );

  // Insert
  await db.insert('users', {'id': 1, 'name': 'Harshil'});
  // Query
  final result = await db.query('users');
  print(result);
}

üëâ Best for complex apps (chat, e-commerce, offline-first apps).
