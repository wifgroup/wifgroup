Day 17: Firebase Setup in Flutter

Connect your Flutter app to a real backend for authentication, databases, and more. üöÄ

üß† Why Firebase?

Yesterday, we explored Provider state management, making our apps scalable. Today, we‚Äôll set up **Firebase** in our Flutter project ‚Äî the backend that powers authentication, real-time databases, and cloud storage. Firebase is Google's comprehensive mobile and web backend that offers easy integration with Flutter.

Firebase offers services like:

  • **Authentication** (Email/Password, Google, Phone)
  • **Firestore** (Real-time NoSQL database)
  • **Cloud Storage** (Store files & images)
  • **Push Notifications**

üì¶ Step 1: Create a Firebase Project

The first step is to create a new project in the Firebase Console.

  1. Go to the Firebase Console.
  2. Click **Add project** and give it a name (e.g., `flutter_app`).
  3. Follow the prompts, and click **Create project**.

üì± Step 2: Add Flutter App to Firebase

Next, connect your Flutter project. You'll need to do this for each platform you're targeting (Android, iOS).

  1. From the Firebase project overview, click the **Add app** icon (iOS or Android).
  2. Enter your app's **Bundle ID** (for iOS) or **applicationId** (for Android). You can find this in `android/app/build.gradle` for Android.
  3. Download the configuration file (`google-services.json` for Android, `GoogleService-Info.plist` for iOS).
  4. Place the file in the correct location:
    • **Android:** `android/app/`
    • **iOS:** `Runner/` in Xcode

üìå Step 3: Configure Flutter Project

You'll need to add the Firebase dependencies to your Flutter project and configure the native platform files.

Add Dependencies

Add these to your `pubspec.yaml` file:

pubspec.yaml 

dependencies:
  flutter:
    sdk: flutter
  firebase_core: ^2.15.0
  firebase_auth: ^5.1.0
  cloud_firestore: ^5.6.0

Then, run `flutter pub get`.

Android Configuration

Add the Google services classpath to `android/build.gradle`:

android/build.gradle 

classpath 'com.google.gms:google-services:4.3.15'

And apply the plugin in `android/app/build.gradle`:

android/app/build.gradle 

apply plugin: 'com.google.gms.google-services'

iOS Configuration

Drag the `GoogleService-Info.plist` file into your Xcode project's `Runner` directory and run `pod install` in the `ios` directory.

üìå Step 4: Initialize Firebase in Flutter

Before you can use any Firebase services, you must initialize it in your `main` function. We recommend using the `flutterfire` CLI tool to generate `firebase_options.dart` for you.

main.dart 

import 'package:flutter/material.dart';
import 'package:firebase_core/firebase_core.dart';
import 'firebase_options.dart'; // generated by flutterfire CLI

void main() async {
  WidgetsFlutterBinding.ensureInitialized();
  await Firebase.initializeApp(
    options: DefaultFirebaseOptions.currentPlatform,
  );
  runApp(MyApp());
}

class MyApp extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    return MaterialApp(home: Scaffold(body: Center(child: Text("Firebase Ready!"))));
  }
}

üîë Step 5: Firebase Authentication Basics

To use authentication, you must enable the desired sign-in method in your Firebase console.

Navigate to **Authentication** -> **Sign-in method** -> **Email/Password** -> **Enable**.

üì¶ Step 6: Firestore Basics

Firestore is a flexible, scalable NoSQL cloud database. To get started, navigate to **Firestore Database** in the console and click **Create Database**. You can then add data programmatically.

Add a document to Firestore 

import 'package:cloud_firestore/cloud_firestore.dart';

FirebaseFirestore firestore = FirebaseFirestore.instance;

await firestore.collection('users').add({
  'name': 'John Doe',
  'email': 'john@example.com',
});
